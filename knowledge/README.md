# Click2Call Knowledge Base

This folder contains internal operational notes for answering emails consistently and accurately. It is intended to sit beside the public website content while storing support guidance, approved wording, internal rules, and reply templates that should not necessarily appear on the public site.

## Folder structure

| Folder | Purpose |
|---|---|
| `services/` | Internal summaries of each service and how to describe it in replies |
| `pricing/` | Canonical pricing notes, quoting rules, and billing guidance |
| `support/` | Troubleshooting, onboarding, escalation, and operational notes |
| `email-replies/` | Ready-to-use reply templates for common customer scenarios |
| `policies/` | Internal rules on claims, promises, sender identity, and sensitive topics |
| `faq-internal/` | Common customer questions not fully covered on the public website |

## Usage notes

Always check that replies remain consistent with the current public website, support portal, and live pricing before sending. When public website wording and internal notes differ, confirm the latest official business position before replying.

A particular point to resolve is the official support email identity. The website content currently includes references to `click2call@bcomservices.com`, while the mailbox now being configured is `support@click2call.com.au`. Until this is standardised, replies should use the address and branding the business explicitly approves.
