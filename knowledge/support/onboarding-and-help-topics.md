# Onboarding and Help Topics

This file lists the main support and setup topics already reflected in the repository help content.

## Common onboarding topics

| Topic | Notes for replies |
|---|---|
| Account activation | Direct customers to the account activation help article or portal instructions. |
| Adding account credit | Explain the steps clearly and confirm whether billing authority is needed. |
| Adding channels | Useful when a customer needs more simultaneous call capacity. |
| Adding users and extensions | Relevant when a business is setting up staff or departments. |
| Adding phone numbers | Relevant for new services or expansion. |
| Number porting | Confirm existing number details and expected provisioning process before giving timelines. |
| Desk phones | Confirm supported devices before promising compatibility. |
| Softphones | Useful for customers wanting to use mobiles or computers instead of desk phones. |
| Business hours | Relevant for routing, after-hours handling, and AI receptionist behaviour. |
| Ring groups and call flow | Useful for teams, departments, and call routing logic. |
| Voicemail and call recordings | Explain clearly what the customer can access and where. |
| AI receptionist setup | Use for businesses wanting 24/7 automated answering or call qualification. |

## Support tone guidance

Replies should be calm, specific, and practical. Use short explanations and, where useful, give the customer the next concrete step. If a request spans several setup steps, break the reply into a short sequence instead of sending a very long explanation.

## Escalate when needed

Escalate internally when a customer reports a service outage, porting failure, billing dispute, legal concern, or any request that would require access to account-level data not already available in the email thread.
