# Services Overview

This file provides a plain-language summary of Click2Call services for use in customer email replies.

## Core positioning

Click2Call provides Australian business telephony services built around cloud-based calling, flexible setup, and simple month-to-month pricing. The main themes presented across the public website are fast setup, no required hardware, no lock-in contracts, and support for businesses across Australia.

## Main services

| Service | Internal summary for replies |
|---|---|
| Cloud PBX | A hosted business phone system that lets businesses make and receive calls without traditional on-site PBX hardware. |
| VoIP phone system | Internet-based business calling that can work on mobiles, laptops, desktops, and compatible desk phones. |
| AI Receptionist | An automated answering layer designed to handle inbound calls, capture caller intent, and route calls appropriately. |
| AI Voice Tools | AI-supported voice features such as transcription, analysis, and call-related productivity tools. |
| SIP Trunks | A way for businesses to connect compatible phone systems to internet telephony services. |
| Microsoft Teams Calling | Calling capability that allows businesses to make and receive external phone calls through Microsoft Teams. |

## Useful wording for replies

When replying to enquiries, use clear and direct language. It is generally appropriate to describe the service as suitable for Australian businesses that want a flexible cloud phone system, want to avoid traditional hardware complexity, or need a modern alternative to legacy office phones.

Avoid over-promising features that have not been confirmed in the public website or account documentation. If a customer asks about a specific workflow, handset, integration, porting timeline, or billing outcome, confirm against current documentation or internal policy before giving a firm answer.
